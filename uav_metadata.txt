METADATA ANALYSIS
General
GpsStatus=”RTK”/AltitudeType="RtkAlt" -> generated by DJI drones equipped with Real-Time Kinematic (RTK) positioning, where the vertical position is referenced to the WGS84 ellipsoid - good!
GpsLatitude/Longitude -> drone body position not optical center of camera sensor
AbsoluteAltitude -> PRESUMABLY ELIPSOIDAL height 
RelativeAltitude -> height above takeoff point - this won’t do (it would do for local frame)
ImageSource -> if needed for camera intrinsics (RGB uses the WideCamera)


Gimbal orientation fields (where camera is pointing)-> defined relative to the NED frame: North-East-Down, euler sequence is intrinsic ZXY
GimbalYawDegree -> absolute or relative??
GimbalPitchDegree -> negative means downward
GimbalRollDegree -> usually 0 but include either way
RTK quality fields see rtk metadata
RtkFlag
0 = no satellite signal
15 = no position solution
16 = single-point solution, meter-level
34 = float/decimeter level
50 = integer ambiguity solution, centimeter-level
StdLon / StdLat / StdHgt -> standard deviation
Surveying / mapping suitability
drone-dji:SurveyingMode="1" -> can be mapped, if 0 then accuracy not guaranteed
Laser rangefinder (LRF) fields
drone-dji:LRFStatus="Normal" 
LRFTargetDistance -> distance in meters from camera to ranged target point, valid when above is normal
LRFTargetLon / Lat -> lon/lat of ranged target point
LRFTargetAlt -> height of ranged target point relative to takeoff point
LRFTargetAbsAlt -> target’s absolute ellipsoidal alt
